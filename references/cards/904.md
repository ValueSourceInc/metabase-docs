## Description

业务口径 1. row_type = order_level 表示可以明确归属到订单和 SKU 的已售经营行。它来自 Amazon settlement 的订单级 revenue/cost 数据，并补充 SKU 单件生产成本、单件物流成本和广告分摊成本。 2. row_type = uncovered_platform_cost 表示无法明确归属到订单/SKU 的平台级费用。这些费用不强行分摊到 SKU，而是单独保留。 3. product_sales_usd 是商品销售本金汇总，只表示商品销售额，不扣 Amazon 费用、广告或产品成本。 4. amount_usd 是订单/SKU 层面的经营金额，已经包含 settlement 原始金额和 allocated_ad_cost_usd（广告分摊成本已内嵌在 amount 中）。allocated_ad_cost_usd 列仅作参考分解。 5. gross_sold_qty 是购买数量；sellable_return_qty 是可售退货数量；cost_recognized_qty = max(gross_sold_qty - sellable_return_qty, 0)。 6. avg_landed_unit_cost_usd = avg_production_unit_cost_usd + avg_shipping_unit_cost_usd。 7. product_cost_usd = cost_recognized_qty × avg_production_unit_cost_usd。 7a. shipment_cost_usd = cost_recognized_qty × avg_shipping_unit_cost_usd。 8. net_profit_usd： - order_level 行：net_profit_usd = amount_usd - product_cost_usd - shipment_cost_usd 注：amount_usd 已包含广告成本，allocated_ad_cost_usd 列仅供参考分解，不在利润公式中重复扣除。 - uncovered_platform_cost 行：等于平台级费用金额。 9. missing_unit_cost = true 表示有成本确认数量但缺少 SKU 单件成本数据。
