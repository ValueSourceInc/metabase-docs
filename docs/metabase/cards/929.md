## Description

这张图是“全摊薄后的 SKU 累计已售经营表现” 它基于 #928 全摊薄已售经营表现底表：先把原来没有 SKU 的平台费用，按业务原因分到各个 SKU，再计算每个 SKU 的累计销售额、累计全摊薄利润(fully_loaded_net_profit)和累计成本。 分摊逻辑（简单版）： - 仓储费：按库存占用体积来分，库存越多、体积越大，分到越多。 - 入仓配置费：按入库数量来分，入库越多，分到越多。 - 移除/销毁费：按损坏/缺陷退货数量来分（#733 退货表），损坏退货越多，分到越多。 - 订阅费、促销活动费、Vine 费用：按销售额来分，销售额越高，分到越多。 使用注意： - 筛选单个 SKU 或多选 SKU 时，看到的是这些 SKU 原本应承担的平台费用份额。 - 不建议用“排除某些 SKU”的方式看剩余整体，因为被排除 SKU 原本分到的平台费也会一起被排除，不会自动重新分摊给剩余 SKU。 - 如果要看和店铺总经营表现完全对齐的整体趋势，应使用不排除 SKU 的全量口径。
